## Key Decisions Already Made (Do Not Re-Debate)

| Decision | Value | Source |
|---|---|---|
| Architecture style | DDD Modular Monolith | `04_architecture/01_architecture.md` |
| Session type | ALL sessions are HYBRID (no session type selector) | `02_features/05_sessions.md` |
| Manager authority | Derived from `user_hierarchy` table — NOT a Keycloak role | `04_architecture/01_architecture.md` |
| Compliance owner | Assignment Engine — NOT a standalone module | `04_architecture/03_modules.md` |
| Auth model | Keycloak SSO + PKCE; JWT in memory ONLY, never localStorage | `02_features/01_authentication.md` |
| Hierarchy depth | depth = 1 (direct reports only; no transitive hierarchy in Phase 1) | `04_architecture/03_modules.md` |
| File storage | OneDrive only — LMS stores metadata only, never file bytes | `04_architecture/04_integrations.md` |
| Search engine | PostgreSQL full-text search only — no Elasticsearch | `04_architecture/03_modules.md` |
| Export formats | Excel (.xlsx) and PDF only — no CSV | `04_architecture/01_architecture.md` |
| Database | Single PostgreSQL DB, single schema, UUID PKs everywhere | `04_architecture/01_architecture.md` |
| Token storage | Memory only — never DB, localStorage, sessionStorage, or cookies | `02_features/01_authentication.md` |
| APISIX role | JWT validation + routing only — no role checks at gateway | `04_architecture/01_architecture.md` |
| Attendance tracking | Always enabled — no toggle to disable | `02_features/05_sessions.md` |
| Assignments deletion | Never hard-deleted — CANCELLED status used | `04_architecture/01_architecture.md` |
| Audit writes | Always async — audit failures NEVER block originating operations | `02_features/08_audit.md` |
| PES integration | PES pulls from LMS — LMS NEVER pushes to PES | `04_architecture/04_integrations.md` |
