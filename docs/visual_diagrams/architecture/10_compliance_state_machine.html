Critical Rule: Completing an OVERDUE assignment late sets compliance_state = NON_COMPLIANT permanently. The only way to become COMPLIANT again for that training is through a new assignment (recertification), which starts as PENDING.

State Definitions

StateDefinitionDB ConditionPES API Returns
PENDING Assignment exists and is active. Employee has not yet completed, and due_date has not passed. assignment_status IN (ASSIGNED, IN_PROGRESS) AND due_date >= today OR due_date IS NULL compliance_state = PENDING
COMPLIANT Assignment completed on time. Or completed with no due date (no deadline = always compliant on completion). assignment_status = COMPLETED AND (completed_at <= due_date OR due_date IS NULL) compliance_state = COMPLIANT
NON_COMPLIANT Assignment overdue (due date passed without completion) OR completed after the due date. Once NON_COMPLIANT, this assignment record cannot become COMPLIANT. assignment_status = OVERDUE, OR (assignment_status = COMPLETED AND completed_at > due_date) compliance_state = NON_COMPLIANT

Recertification Cycle

StepWhat Happens
1. COMPLIANT reachedtraining_completions row created with completed_at and expires_at (= completed_at + validity_period_days)
2. Completion expiresDaily BG job detects training_completions.expires_at < today
3. New assignment createdAssignment Engine creates new assignment (source = RECERTIFICATION or MANDATORY). compliance_state of OLD assignment stays COMPLIANT.
4. New PENDING rowNew compliance_status row created for the new assignment with state = PENDING
5. Cycle repeatsEmployee must complete the training again. If they do so on time → COMPLIANT again. If not → NON_COMPLIANT.

Note: The OLD compliance_status row (COMPLIANT) is retained. Only the NEW recertification assignment creates a new PENDING row. Both rows exist simultaneously — reporting distinguishes them by assignment_id.

RuleDetail
