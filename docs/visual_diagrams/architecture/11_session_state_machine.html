Session Status Definitions

StateMeaningWho Sets ItAllowed Changes
SCHEDULED Session is created and upcoming. Participants can be added. Nominations can be submitted and reviewed. System (on Admin creates session) Can be rescheduled (date/time/venue), cancelled, or completed. Participants can be added or removed.
COMPLETED Session occurred. Attendance has been finalized. Completion events fired for all ATTENDED participants. Terminal — locked. Admin (manual action) None. No structural changes allowed. Attendance records are permanent.
CANCELLED Session did not occur. No completion generated. All participants notified. Retained in history permanently. Terminal. Admin None. Cannot be rescheduled or completed after cancellation.

Teams Link Status (Sub-state Within SCHEDULED)

teams_link_statusMeaningHow It Gets SetAction Required?
AUTO_CREATED Teams meeting was created successfully via Graph API. Link stored in sessions table. All participants can join from LMS. System (on session create or reschedule success) None
PENDING_MANUAL Teams meeting creation or update failed. No meeting link available yet. Session is valid — meetings are not required for sessions to exist. System (on Teams API failure) Admin must provide manual link before session goes live
MANUAL_PROVIDED Admin has manually entered a Teams/meeting link. Participants can use it to join. Admin (manual form update) None — but Admin should verify link is correct

Rescheduling Rules

RuleDetail
Only SCHEDULED sessions can be rescheduledCOMPLETED and CANCELLED sessions cannot be rescheduled. System returns 409 if attempted.
What can be changed on rescheduledate/time (start + end), venue. Facilitator can also be updated. Max participants can be updated.
Teams update on rescheduleSystem attempts to update the Teams meeting via Graph API. If it fails: teams_link_status = PENDING_MANUAL. Admin must update the link manually.
Re-notificationAll INVITED and CONFIRMED participants receive a reschedule notification. NOMINATED (pending review) participants do NOT receive re-notification.
Audit eventSESSION_RESCHEDULED audit event emitted with old/new datetime, old/new venue_id, updated_by.

Participant Enrollment Status Definitions

StatusMeaningWho Sets ItCounts Against max_participants?
INVITED Admin directly added this participant. Considered fully enrolled. Receives session notifications. Admin Yes
NOMINATED Manager has nominated this participant. Awaiting Admin review. Employee is notified of nomination. Manager (within nomination window) Yes — counted to prevent over-booking during review
CONFIRMED Admin has confirmed the manager's nomination. Participant is fully enrolled. Same as INVITED. Admin (confirms nomination) Yes
REJECTED_NOMINATION Admin rejected the manager's nomination. Employee not enrolled. Both Manager and Employee notified with reason. Admin (rejects nomination) No — removed from count on rejection

Attendance Status Definitions

StatusMeaningWho/What Sets ItDrives Completion?
ATTENDED Participant attended the session (online via Teams or offline in-person). Only ATTENDED drives training resource completion. Teams auto-fetch (TEAMS_AUTO), Admin/HR manual (ADMIN_MANUAL), or Admin override (ADMIN_OVERRIDE) Yes — triggers resource_progress → COMPLETED
NOT_ATTENDED Participant did not attend. No completion triggered. Assignment for session resource remains incomplete. Teams auto-fetch, Admin/HR manual, or default for unmarked participants No — assignment remains ASSIGNED or IN_PROGRESS

Nomination Window Rules

RuleDetail
Nomination window is optionalIf not configured, only Admin can add participants directly. No nomination flow available.
nomination_open_at and nomination_close_at must both be before start_timeEnsures nominations are reviewed before the session begins.
Manager can only cancel while window is openOnce nomination_close_at passes, the Manager cannot withdraw. Admin must reject it.
Admin can add participants directly even when nomination window is activeDirect adds bypass the nomination flow entirely.
Duplicate enrollment blockedSame user cannot be both INVITED and NOMINATED. Returns 409 DUPLICATE_ENROLLMENT.
Capacity cap applies to NOMINATED + CONFIRMED + INVITED combinedIf max_participants is reached, further direct adds and nomination confirmations are blocked. SESSION_AT_CAPACITY error.
