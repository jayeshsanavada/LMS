Failure Isolation Rule

PrincipleDetail
No integration failure blocks a core user operationAll 6 integrations are asynchronous or non-blocking. A Teams API failure does not stop session creation — it sets teams_link_status = PENDING_MANUAL. A Zoho failure does not stop user login — last-known data is used.
Retry policy for outbound calls3 automatic retries with 5-minute intervals before marking the job as FAILED and alerting Admin. Failed jobs recorded in integration_jobs table.
PES is read-only from LMS perspectivePES reads compliance_status. LMS does not write to PES. Every PES call is audited in audit_logs regardless of result.
