Internal Routes — Why They Are Not in APISIX

RoutePurposeWhy loopback only
/api/v1/internal/notifications Modules fire notification events to the Notifications module without going through the public gateway These are service-to-service calls within FastAPI. Exposing them via APISIX would create a publicly accessible endpoint that could trigger notifications without proper user context.
/api/v1/internal/audit All state-changing operations write audit events asynchronously via this route Audit writes must never be callable from outside the server. Loopback ensures only the application itself can emit audit events.
