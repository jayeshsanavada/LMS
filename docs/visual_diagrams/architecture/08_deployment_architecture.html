Availability When External Systems Are Down

System DownLMS Behavior
KeycloakLogin blocked. Existing valid sessions last until token expiry (~30 min). Users mid-session can complete current operations.
Zoho HRContinue with last-synced user data. Sync job logged as FAILED in integration_jobs. No user-facing error shown.
Employee DBUse last-known hierarchy. Manager-scoped operations may be degraded (stale reports list). No user-facing error shown.
Microsoft TeamsSession record persists. teams_link_status = PENDING_MANUAL. Admin provides manual meeting link. Attendance entered manually after session.
OneDriveResource file access blocked. Training metadata (title, description, structure) still accessible. New uploads blocked. 3 auto-retries then Admin alerted.
PESPES calls fail on PES side. No LMS impact. LMS has no dependency on PES being available. Every call that does reach LMS is still audited.

Security Architecture

ConcernMechanism
JWT storageMemory only — Keycloak JS Adapter. Never written to localStorage, sessionStorage, cookies, or database.
Double JWT validationAPISIX validates signature + expiry + aud claim. FastAPI re-validates independently. Two independent checks.
Internal route protectionInternal routes (/api/v1/internal/*) are not registered in APISIX. Only accessible via loopback from within the same process.
PES isolationPES M2M uses a separate APISIX route group with a 100 req/min rate limit and optional IP allowlist.
Role enforcement locationAPISIX does NOT check roles. All RBAC and hierarchy-based access control is enforced in FastAPI middleware.
