State Definitions

StateMeaningWho/What Sets ItCan Transition To
ASSIGNED Assignment exists, employee has not yet started any resource System (auto), Manager, Admin IN_PROGRESS, OVERDUE, CANCELLED
IN_PROGRESS Employee has opened at least one resource but has not completed all required ones System (on first resource access) COMPLETED, OVERDUE, CANCELLED
COMPLETED All required resources completed. Terminal state — cannot be cancelled. System (on last resource completion or session attendance = ATTENDED) None (terminal). May trigger recertification assignment.
OVERDUE due_date has passed and assignment is not yet completed. Escalation notifications begin. Daily background job (overdue detection) COMPLETED (late), CANCELLED
CANCELLED Assignment cancelled before completion. Retained permanently in history. Terminal state. Admin (any non-COMPLETED assignment) or Manager (assignments they created for direct reports) None (terminal)

Duplicate Prevention Rule

SituationBehavior
Same user + same training with status IN (ASSIGNED, IN_PROGRESS, OVERDUE)409 DUPLICATE_ASSIGNMENT error. New assignment blocked. Existing assignment ID returned.
Same user + same training with status COMPLETEDAllowed. Recertification use case. New assignment created normally.
Same user + same training with status CANCELLEDAllowed. Previous cancelled record retained. New assignment created.

State Definitions

StateMeaningWho/What Sets It
PENDING Request submitted. Awaiting manager or admin decision. System (on employee request submission)
APPROVED Request approved. An ASSIGNED assignment is auto-created with source = SELF_APPROVED. Manager (direct manager via hierarchy) or Admin
REJECTED Request rejected. No assignment created. Rejection reason stored. Manager or Admin
EXPIRED No decision made within the configured expiry window. Request closed automatically. Background job (checks expires_at < now)

Approval Rules

RuleDetail
Who can approveManager (direct manager from user_hierarchy at time of request) or Admin. Admin is the Phase 1 fallback if the manager is unavailable.
approver_user_id is immutableSet at request creation. Not updated if the user's manager changes after the request is submitted. Admin fallback covers this case.
One PENDING request per trainingSame user cannot have two PENDING requests for the same training simultaneously. Returns 409 if duplicate PENDING exists.
ExpiryDefault 30 days. Configurable in admin_settings. Background job runs daily to check expires_at < now().
Re-submissionExpired requests can be re-submitted. Previous expired record is retained in history.

When the Approval Flow Is Used

ConditionFlow
training_items.requires_approval = falseEmployee directly self-enrolls. No request created. Assignment source = SELF_ENROLLED.
training_items.requires_approval = trueEmployee must go through request flow. Assignment only created after approval. Source = SELF_APPROVED.
Mandatory / Manager / Admin assignmentsApproval flow is NEVER used. Assignments created directly.
