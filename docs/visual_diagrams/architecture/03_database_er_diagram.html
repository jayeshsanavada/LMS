Section 1 Key Relationships

RelationshipDetail
users → user_hierarchySelf-referencing via manager_user_id. Only depth = 1 (direct reports) is used for hierarchy-scoped access. is_current = true marks the active row.
assignments → compliance_statusEach assignment drives one compliance_state record per user+training combination. Updated nightly by the Compliance Eval scheduled job.
training_items → sessionsILT-type trainings have sessions. SELF_PACED trainings do not. A training_item can have multiple sessions across time.
assignments → assignment_requestsOptional — only present when an approval workflow is required. Controlled by mandatory_assignment_rules.requires_approval.
resource_filesStores OneDrive file references only (external_file_id, reference_url). Never stores file bytes. Actual files live in OneDrive.

Section 2 Key Relationships

RelationshipDetail
notifications → notification_delivery_logEach delivery attempt is logged. Failed attempts accumulate here for retry by the hourly dead-letter retry job.
audit_logsINSERT-only table — no UPDATE or DELETE operations ever. legal_hold = true records survive the weekly retention purge.
audit_write_failuresDead-letter for failed async audit writes. Retried hourly. Surfaced in Admin Dashboard.
report_exportsRow created when export is requested. expires_at = 24h. Cleaned up by daily Export File Cleanup job.
admin_settingsKey-value config store. Audit-trailed via admin_settings_history. Used by Notifications and Assignment Engine for configurable behavior.

Section 3 Key Relationships

RelationshipDetail
integration_jobs → integration_job_logsEach sync job run creates a job record and appends logs. Failed runs stay in integration_jobs with status FAILED for Admin visibility.
integration_health_statusUpdated every 15 minutes by the Integration Health Check job. One row per integration_name. Admin Dashboard reads this table.
resource_assessments → assessment_attemptsAssessment config (pass_percent, max_attempts) lives in resource_assessments. Each attempt is a separate row. Score and status are immutable after completion.
resource_progressTracks per-resource completion progress (0–100%). Updated as user consumes content. Drives overall training_completions.
user_capabilitiesSynced from Employee DB. Used by mandatory_assignment_rules to scope assignments to users with specific capability tags.
