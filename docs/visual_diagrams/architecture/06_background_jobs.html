Scheduled Jobs — Run on Fixed Schedule (APScheduler)

JobScheduleOwning ModuleWhat it does
Zoho HR SyncDaily 02:00IntegrationsPulls HR data from Zoho (name, dept, status, joining_date). Updates users table. 3 retries on failure.
Employee DB SyncDaily 02:30IntegrationsPulls org hierarchy and project allocations. Replaces user_hierarchy and user_project_allocations for changed records.
Integration Health CheckEvery 15 minIntegrationsPings all 6 external systems. Updates integration_health_status. Sets DEGRADED / DOWN states visible in Admin UI.
Compliance / Mandatory Rule EvaluationNightly 03:00Assignment EngineRe-evaluates all mandatory rules. Creates new assignments for newly-in-scope users. Updates compliance_status for all users.
Overdue Detection + EscalationDailyAssignment EngineFlags assignments past due_date as OVERDUE. Sends escalation notifications to managers for direct reports.
Approval Expiry CheckDailyAssignment EngineCancels assignment_requests past expires_at. Updates status to EXPIRED. Notifies requester.
Due-Date Reminder DispatchDailyNotificationsSends upcoming due-date reminders to employees. Configurable lead time in admin_settings.
Session Reminder DispatchHourlyNotificationsSends session start reminders to enrolled participants. Fires N hours before session start_time.
Notification Record PurgeDailyNotificationsDeletes notifications older than 90 days retention window from the notifications table.
Notification Dead-Letter RetryHourlyNotificationsRe-attempts delivery of records in notification_delivery_log with status FAILED. Max 3 retries total.
Export File CleanupDailyReportingDeletes report_exports files past expires_at (24h retention). Removes from OneDrive if stored there.
Audit Log Retention PurgeWeeklyAuditArchives audit_logs older than 5-year retention. Skips records with legal_hold = true.
Audit Dead-Letter RetryHourlyAuditRe-attempts writing events from audit_write_failures. Failures here are surfaced in Admin Dashboard.

Event-Driven Jobs — Triggered by Business Events (FastAPI Background Tasks)

Trigger EventBackground TaskFailure Handling
Assignment created (any source)Dispatch notification to assigned employee. Write audit event.Dispatch failure → notification_delivery_log
Session createdCall Teams Graph API to create meeting. Store teams_meeting_id in sessions.teams_link_status = PENDING_MANUAL. Admin provides manual link.
Session rescheduledUpdate Teams meeting via Graph API. Re-notify all enrolled participants.Notify failure → notification_delivery_log. Teams failure → PENDING_MANUAL.
Session cancelledCancel Teams meeting. Notify all enrolled participants.Notify failure → notification_delivery_log
Session endedPull attendance report from Teams Graph API. Populate session_attendance records.Attendance entries empty. Admin enters manually.
Approval request expiredUpdate assignment_request.status to EXPIRED. Notify requester.Notify failure → notification_delivery_log
Training completion expiredAuto-create a new recertification assignment for the employee.Compliance state updated to NON_COMPLIANT if recertification not started.
User enters NON_COMPLIANT stateSend escalation notification to employee and their manager.Notify failure → notification_delivery_log
Any state-changing operationWrite structured event to audit_logs asynchronously.Write failure → audit_write_failures. NEVER blocks the originating API call.

Key Architecture Rules

RuleDetail
Worker process is SEPARATERuns as a separate OS process — cannot share in-memory state with the FastAPI API server
Audit writes never blockAudit failures go to audit_write_failures dead-letter — the originating API operation is never blocked or rolled back
Notifications always asyncNotification dispatch is never inline in an API response — always dispatched via Worker queue
All jobs are idempotentRe-running any scheduled or event-driven job produces the same result without duplication
Dead-letter visibilitynotification_delivery_log, audit_write_failures, integration_jobs — all surfaced in Admin UI
