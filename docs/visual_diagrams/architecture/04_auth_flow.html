Security Rules

aud claim must match LMS Keycloak client ID — wrong audience = 401APISIX
Manager is NOT a Keycloak role — derived from user_hierarchy.manager_user_idFastAPI middleware
Hierarchy depth = 1 — direct reports only, no transitive scopeFastAPI service layer
PES M2M uses separate APISIX route group — 100 req/min rate limitAPISIX (PES route group)
Every PES call logged as PES_API_ACCESSED audit eventFastAPI — Audit module (async)
No passwords stored in LMS — Keycloak is sole auth providerArchitecture constraint

Sequence Flow Summary

SequenceWho InitiatesKey StepsNotes
1 — PKCE LoginBrowser (user clicks login) Browser → Keycloak → Azure AD → Keycloak → JWT to Browser JWT never touches localStorage or DB. Stored in Keycloak JS Adapter memory only.
2 — API RequestBrowser (any LMS action) Bearer JWT → APISIX validates → FastAPI re-validates + DB lookup → Response Double JWT validation. Manager scope derived from user_hierarchy, not Keycloak roles.
3 — Token RefreshKeycloak JS Adapter (automatic) Silent iframe refresh → new JWT → old discarded Transparent to user. Happens before token expiry. No session interruption.
4 — PES M2MPES system (external on-prem) Client credentials → M2M JWT → APISIX PES route group → FastAPI → compliance read Every call rate-limited (100/min) and audited. INTEGRATION scope only — cannot access other APIs.
