Component Reference — Layer by Layer
Presentation Layer
Browser
ReactJS SPA — Single-page application served to the browser. Users interact with all LMS features (catalog browsing, assignments, reports, admin) through this UI. No server-side rendering; all data fetched from FastAPI via REST.

Keycloak JS Adapter — Client-side library embedded in the SPA. Handles the full PKCE login flow silently, stores the JWT in browser memory only (never localStorage or cookies), and auto-refreshes tokens before they expire without the user noticing.
Gateway Layer
APISIX
APISIX API Gateway — The only entry point from the internet into the backend. Every request — from browsers and external services — must pass through here.

It performs four things before forwarding to FastAPI:
  • Validates JWT signature, expiry, and aud claim
  • Terminates TLS (port 443)
  • Injects X-Correlation-ID header for request tracing
  • Applies rate limiting (100 req/min on PES M2M routes)
It does not check roles — that is FastAPI's job.
API Layer
FastAPI / Python
FastAPI — The backend application server. Runs on port 8000 (internal only, not reachable from outside). Handles all business logic after APISIX forwards the request.

Responsibilities:
  • Re-validates JWT and extracts user identity
  • Enforces role-based access (ADMIN / HR / EMPLOYEE)
  • Enforces hierarchy scope — managers can only see their direct reports (depth = 1), derived from the user_hierarchy table, not from Keycloak
  • Delegates work to the 16 application modules
  • Fires background tasks asynchronously for notifications, audit writes, and Teams meetings
Data Layer
Persistence
PostgreSQL — Single relational database holding all structured LMS data: users, training items, assignments, sessions, compliance records, notifications, audit logs, and settings.

OneDrive — Binary file storage for training resources (PDFs, videos, PPTXs). LMS never stores file bytes in PostgreSQL — only metadata (external_file_id, reference_url). Streaming URLs are re-generated on each access if expired.
Worker Layer
Separate OS Process
Runs as a completely separate server process from FastAPI — they share no memory. If the Worker crashes, the API server keeps serving users. If FastAPI crashes, scheduled jobs keep running.
Contains two types of jobs: Scheduled jobs (APScheduler — run on a clock) and Event-driven jobs (FastAPI background tasks — fired by application events, not a schedule). All jobs are idempotent — re-running produces the same result with no duplication. Failed jobs are written to dead-letter tables (audit_write_failures, notification_delivery_log) and are visible in the Admin dashboard.
Scheduled Jobs — APScheduler (run on a timer)
JobScheduleWhat It Does
Zoho HR SyncDaily 02:00Pulls employee name, dept, status, joining date from Zoho. Zoho is source of truth — overwrites LMS unless field has an Admin override.
Employee DB SyncDaily 02:30Pulls manager hierarchy, capabilities, and project allocations. This is the source of truth for who reports to whom — not Keycloak.
Integration Health CheckEvery 15 minPings each external system (Zoho, Teams, OneDrive, Keycloak). Updates integration_health_status to HEALTHY / DEGRADED / DOWN.
Compliance EvaluationNightly 03:00Evaluates all mandatory assignment rules. Creates new assignment records for employees who qualify. Updates compliance_status to PENDING / COMPLIANT / NON_COMPLIANT.
Overdue Detection + EscalationDailyFinds assignments past due date. Updates status to OVERDUE. Triggers escalation notifications to manager if threshold is exceeded.
Approval Expiry CheckDailyFinds pending approval requests past their expiry date. Marks them EXPIRED and fires a notification to the employee.
Due-Date Reminder DispatchDailySends reminder notifications to employees whose assignment due date is approaching (configurable days-before window from Admin settings).
Session Reminder DispatchHourlySends reminder notifications to session participants before upcoming sessions (configurable hours-before window).
Notification Record PurgeDailyDeletes notification records older than 90 days to keep the table size manageable.
Notification Dead-Letter RetryHourlyRetries delivery of failed notifications sitting in notification_delivery_log (up to 3 attempts with exponential backoff).
Export File CleanupDailyDeletes export files (CSV, Excel) from storage that are older than 24 hours.
Audit Log Retention PurgeWeeklyDeletes audit records older than 5 years (unless flagged with legal_hold = true).
Audit Dead-Letter RetryHourlyRetries failed audit writes sitting in audit_write_failures. Failed audit writes never blocked the original operation — they are caught here.
Event-Driven Jobs — Background Tasks (fired by application events)
Trigger EventWhat Runs in Background
Assignment createdSend ASSIGNMENT_CREATED notification to the employee. If delivery fails, written to dead-letter for hourly retry.
Session createdCall Microsoft Teams Graph API to create an online meeting. Store teams_meeting_id and link in the sessions table. If Teams is down, set teams_link_status = PENDING_MANUAL.
Session rescheduledCall Teams Graph API to update the meeting time. Re-send notifications to all confirmed participants with the new time.
Session cancelledCall Teams Graph API to cancel the meeting. Send SESSION_CANCELLED notification to all participants.
Session end time passesCall Teams Graph API to pull the attendance report. Populate session_attendance with who joined online. If Teams is down, attendance must be entered manually by Admin.
Approval request expiresUpdate assignment_requests.status = EXPIRED. Send APPROVAL_EXPIRED notification to the employee so they can re-request.
Training completion expiresCreate a new recertification assignment for the employee so they must redo the training before the validity window closes.
Compliance state changes to NON_COMPLIANTIf configured threshold is exceeded, send escalation notifications to the employee's manager and HR.
Any state-changing API operationAsync write to audit_logs. This never blocks the API response — if the write fails it goes to audit_write_failures dead-letter and is retried hourly.
Application Layer — 16 Modules
DDD Bounded Contexts
Each module owns its own tables and exposes services only through its public API — no module may write directly to another module's tables. All 16 modules share the single PostgreSQL database.
Core Modules — Primary Business Value
M1 User ManagementManages employee profiles, org hierarchy, probation status. Syncs data from Zoho HR and Employee DB. Single source of identity within LMS.
M2 Training ManagementManages the training catalog: courses, learning paths, curricula. Handles versions, resource uploads to OneDrive, certificates, and completion rules.
M3 Assignment EngineOwns all assignment logic: mandatory rule evaluation, compliance tracking, approval workflow, probation gates. This is the only module that may write compliance state.
M4 SessionsManages instructor-led training sessions: scheduling, venue/facilitator booking, participant nominations, Microsoft Teams meeting creation, and attendance tracking.
Supporting Modules — Enable Core Flows
M5 NotificationsDelivers in-app and email notifications. Always async — never blocks API responses. Manages user preferences and dead-letter retry for failed deliveries.
M6 ReportingRead-only module. Generates dashboards and downloadable reports (compliance, completion, sessions). No data modification — only reads from other modules' tables.
M7 AuditINSERT-only audit log for all state changes. Async writes so failures never block operations. 5-year retention with legal-hold support. Dead-letter for failed writes.
M8 IntegrationsManages connections to external systems: Zoho HR, Employee DB, Microsoft Teams, OneDrive. Owns integration job records and health status. Exposes PES compliance API.
M9 MigrationPhase 2 — Deferred. Will handle one-time migration of historical training data from the legacy system.
Platform Modules — Reusable Capabilities
M10 AuthenticationWraps Keycloak JWT validation. Provides the current-user context (identity + role) to all other modules.
M11 AuthorizationEvaluates role + hierarchy-based access rules. Derived from user_hierarchy (not Keycloak). Manager scope = direct reports only (depth 1).
M12 SearchFull-text and filter-based search over the training catalog. Respects authorization — employees only see published content.
M13 File ManagementHandles upload, delete, and streaming-URL generation for training files on OneDrive via Graph API. LMS never stores file bytes.
M14 Admin ConfigurationKey-value system settings (reminder windows, approval expiry, retention periods). All changes are audit-logged.
M15 API LayerThe FastAPI route definitions and middleware: JWT re-validation, X-Correlation-ID propagation, Pydantic request/response schemas.
M16 Background JobsHosts APScheduler and wires up all event-driven background task handlers. Shared by all modules that need async processing.
